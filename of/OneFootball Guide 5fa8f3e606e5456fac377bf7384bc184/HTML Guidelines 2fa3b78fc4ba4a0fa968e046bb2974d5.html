HTML Guidelines

To ensure that your site's content appears correctly on Onefootball, please familiarise yourself with the HTML guidelines below.

HTML Structure

In order to make sure that our system can correctly parse and display all parts of your articles, please make sure when sending HTML to Onefootball that:

  • There should be proper opening and closing tags around all elements.
  • Basic, fundamental elements like paragraph text and headlines should stay at the root level.
  • Note that all elements not explicitly allowed (see "Supported HTML elements" below) will be ignored.

Supported HTML Elements:

We support the following HTML elements in native Onefootball articles:

  • A text header tag <h1>, <h2>, <h3>, <h4>, <h5>, <h6>.
  • ---> Those tags indicate a hierarchical structure inside of the text (<h1>Section Title</h1>, <h2>Section Subtitle</h2>).
  • ---> All tags will be parsed properly by Onefootball, however only two different styles will be visible in the app: the first one for <h1>, the second one for all other tags.
  • A paragraph <p>.
  • ---> Paragraphs may also be individually wrapped in a divider (<div> tag) or a text (<span> tag). Note that anything else wrapped in such containers will be ignored.
  • ---> Paragraph text can also be styled using italics (<em> tag), bold (<strong> tag), underline (<u> tag), or strikethrough (<strike> tag).
  • A horizontal ruler <hr>.
  • A link <a>.
  • A list, either ordered <ol> or unordered <ul>, in both cases each individual element appears in a <li> tag.
  • An image <img>.
  • ---> Note that the URL of the image should be an absolute file path and not a relative file path.
  • A blockquote <blockquote>.
  • A tweet from Twitter.
  • A YouTube video.
  • A SoundCloud track.
  • ---> Note that in the Onefootball app SoundCloud embeds will be turned into links to the web page on SoundCloud
